Ассоциация, агрегация и композиция — это типы отношений между классами в объектно-ориентированном программировании.

**Ассоциация** — это общее отношение между объектами, которое показывает, что один объект связан с другим. Однако, ни один из объектов не владеет другим.
Пример:
Класс `Student` и класс `Course` связаны ассоциацией: студент посещает курс.

class Student {
    private String name;
}

class Course {
    private String title;
}


**Агрегация** — это слабая форма "часть-целое". Один объект может быть частью другого, но при этом может существовать самостоятельно.
Например, университет (`University`) состоит из факультетов (`Department`), но факультет может существовать сам по себе. Университет не "владеет" факультетом.

class University {
    private List<Department> departments;

    public University(List<Department> departments) {
        this.departments = departments;
    }
}

class Department {
    private String name;
}


Композиция — это более сильная форма "часть-целое". Один объект полностью владеет другим. Если основной объект уничтожается, то уничтожаются и его части.
Например, комната (`Room`) состоит из стен (`Wall`). Если комната уничтожается, стены тоже исчезают.

class Room {
    private Wall wall;

    public Room() {
        this.wall = new Wall();
    }
}

class Wall {
    private String color;
}
